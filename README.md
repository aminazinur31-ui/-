# Lexora Law AI Chat

Теперь это локальный chat-style MVP с реальным серверным подключением к OpenAI через Responses API.

## Что внутри

- `index.html` — интерфейс чата.
- `styles.css` — визуальная система и состояния интерфейса.
- `app.js` — клиентская логика, история вкладок и запросы к локальному backend.
- `server.js` — локальный Node.js сервер, который отдает статику и проксирует `/api/chat` в OpenAI.
- `.env.example` — пример локальной конфигурации.

## Настройка

1. Скопируйте `.env.example` в `.env`.
2. Вставьте свой ключ в `OPENAI_API_KEY`.
3. При необходимости смените `OPENAI_MODEL` или `PORT`.

## Запуск

```bash
node server.js
```

После запуска откройте:

`http://localhost:3000`

## Как это работает

- браузер отправляет историю текущего треда на `/api/chat`
- `server.js` добавляет системные инструкции для Lexora AI
- сервер делает запрос в `POST /v1/responses`
- ответ модели возвращается обратно в чат

## Важно

- ключ OpenAI не уходит в браузер
- финальная юридическая позиция все равно должна проверяться юристом
- если `OPENAI_API_KEY` не задан, интерфейс покажет ошибку настройки
